### 40. Назначение заголовка `Content-Type`

Заголовок `Content-Type` указывает тип содержимого, передаваемого в запросе или ответе. Он помогает серверу и клиенту понять формат данных (например, `application/json` для JSON, `text/html` для HTML), чтобы корректно обработать и отобразить их.

### 41. Назначение заголовка `Accept`

Заголовок `Accept` сообщает серверу, какие типы данных клиент может обрабатывать и предпочитает получить. Например, `Accept: application/json` указывает серверу, что клиент ожидает данные в формате JSON. Сервер использует этот заголовок для определения подходящего формата ответа.

### 42. Назначение `multipart/form-data` в заголовке `Content-Type`

Значение `multipart/form-data` указывается для передачи данных в нескольких частях (партиях), например, при загрузке файлов. Это позволяет разделять части данных и отправлять как текстовые, так и бинарные файлы в одном запросе.

### 43. Как указать `multipart/form-data` в заголовке `Content-Type` с помощью тега `<form>`

Чтобы обеспечить значение `multipart/form-data`, используйте атрибут `enctype="multipart/form-data"` в теге `<form>`, например:

```html
<form action="/upload" method="POST" enctype="multipart/form-data">
  <input type="file" name="file" />
  <button type="submit">Отправить</button>
</form>
```

### 44. Значение заголовка `Content-Type` по умолчанию в форме

По умолчанию `Content-Type` заголовка формы устанавливается в `application/x-www-form-urlencoded`, где параметры отправляются в виде закодированных пар ключ-значение.

### 45. Передача параметров в GET-запросе

Параметры в GET-запросе передаются в строке запроса (query string), которая добавляется к URL после символа `?`. Параметры имеют формат `ключ=значение`, разделяются амперсандом (`&`), например:

```
GET /search?query=nodejs&sort=recent
```

### 46. Передача параметров в POST-запросе

В POST-запросе параметры передаются в теле запроса (body), и их формат определяется заголовком `Content-Type`:

- `application/x-www-form-urlencoded`: пары ключ-значение, разделенные `&`.
- `application/json`: данные передаются в формате JSON.
- `multipart/form-data`: данные передаются в нескольких частях (например, для передачи файлов).

### 47. Понятие JSON

JSON (JavaScript Object Notation) — это текстовый формат для передачи структурированных данных. Он используется для обмена данными между клиентом и сервером. Пример JSON-объекта:

```json
{
  "name": "Иван",
  "age": 25,
  "skills": ["JavaScript", "Node.js", "Express"]
}
```

### 48. Понятие XML

XML (eXtensible Markup Language) — это язык разметки для описания и передачи данных в виде структуры, которая состоит из тегов, как в HTML. XML применяется для обмена данными, где важна их читаемость как человеком, так и программой. Пример XML-структуры:

```xml
<user>
  <name>Иван</name>
  <age>25</age>
  <skills>
    <skill>JavaScript</skill>
    <skill>Node.js</skill>
    <skill>Express</skill>
  </skills>
</user>
```
